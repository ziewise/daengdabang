### 결론 — 수량 절감 vs UI 표시 방식

**수량 줄이기**: 효과 미미 (345 → 340, 1.4%)

**진짜 가치 있는 작업 — UI 카드의 "옵션 묶음" 표시**

데이터는 SKU 단위로 유지하되, **상품 카드에서 같은 모델의 옵션을 묶어 표시**:

```
┌────────────────────────────────────┐
│  [상품 이미지]                     │
│                                    │
│  아페토                            │
│  일체형 오리지널 도넛방석          │
│                                    │
│  ● ● ●  (3 colors)                 │
│  M / L / XL  (3 sizes)             │
│                                    │
│  35,000원 ~                        │
└────────────────────────────────────┘
```

- 같은 모델을 카드 여러 개로 펼쳐서 나오는 혼란 제거
- 사용자는 카드 1개에서 옵션 선택 → 상세 페이지로 이동
- 데이터 구조 변경 불필요 (각 SKU는 별도 row 유지)
- 표시 시점에 `브랜드 + base_name` 으로 그룹핑해서 보여주기

### 적용 시점

- 베스트·신상품·기획전 등 상품 카드 그리드에 일괄 적용
- `main.js` 의 렌더 함수에서 `base_name` 기준 groupBy 로직 추가
